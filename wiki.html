EngineeringPaper Standalone

A standalone fork of EngineeringPaper.xyz — a web app for engineering calculations with automatic unit conversion/checking, plotting, systems of equations, and documentation.

Calculations run in-browser using Pyodide (Python/WebAssembly) with SymPy as the computation engine. This fork requires no server — DOCX export runs via pandoc-wasm and PDF export uses the browser print dialog.

Key Capabilities

  • Math cells with LaTeX input and automatic unit checking/conversion
  • Plotting, piecewise functions, system solvers
  • Data tables, fluid properties, code cells
  • Extreme Value Analysis (EVA) cell — worst-case min/max over parameter ranges
  • DOCX export (pandoc-wasm, native Word equations), PDF export (browser print)
  • Packagable as a single Windows .exe — no installation required

Quick Start

npm install
npm run dev:standalone     # Dev server at localhost:8788 with live reload
npm run build:standalone   # Production build to public/
npm run start:standalone   # Serve public/ at localhost:8788

Windows Executable

Produces a single EngineeringPaper.exe (~330 MB) with Node.js and all assets embedded. No prior software installation required — double-click to launch.

Build the exe

npm run package:win

Runs build:standalone first (rebuilds public/), then packages everything into dist/EngineeringPaper.exe.

Build Steps Separately

npm run build:standalone
npx pkg launcher.cjs --config package.json --targets node20-win-x64 --output dist/EngineeringPaper

Verbose Output

npm run build:standalone
npx pkg launcher.cjs --config package.json --targets node20-win-x64 --output dist/EngineeringPaper --debug

Run

Double-click dist/EngineeringPaper.exe, or from a terminal:

./dist/EngineeringPaper.exe

The server starts at http://localhost:8788 and your default browser opens automatically. Press Ctrl+C to stop.

If port 8788 is already in use, the exe will print an error and exit rather than hanging silently.

Troubleshooting: Port Already In Use

If the exe opens the browser but shows an old version, a previous server process may still be running on port 8788.

netstat -ano | findstr :8788
taskkill /PID <pid> /F

Cell Types

Every row in a sheet is a cell. Cells are inserted via the toolbar or insert menu. Each type has a distinct purpose; results flow from top to bottom — variables defined in earlier cells are available in later ones.

Math Cell

Insert key: 1  |  Icon: +

The primary cell type. Accepts mathematical expressions in LaTeX notation.

  • Assignment: x = 5 [m] — defines a variable with optional units
  • Query: x = — evaluates and displays the result
  • Equation: F = m \cdot a — defines a relationship

Units are automatically converted and checked for dimensional consistency. Number formatting is configurable per cell. An optional annotation column to the right lets you add notes or unit labels. If a range expression is entered, the cell automatically converts to a Plot Cell.

Documentation Cell

Insert key: 2  |  Icon: comment

A rich-text editor (Quill) for adding headings, paragraphs, lists, and formatted notes between calculations. Content is stored as markdown and exported to DOCX/PDF. Spell-check is active when the cell is focused.

Plot Cell

Insert key: 3  |  Icon: smooth line

Renders interactive 2D line plots. Define one or more expressions over a range (e.g., y = x^2 for x from 0 to 10). Supports:

  • Multiple curves on the same axes
  • Up to 4 different y-axis unit groups simultaneously
  • Log scale on either axis
  • Square aspect ratio toggle
  • Copy plot data to clipboard as tab-separated values

Selector Table Cell

Insert key: 4  |  Icon: grid

A lookup table where each row represents a scenario or component variant. The user selects one row via radio button; the selected row's values are injected as variable assignments into the rest of the sheet. Supports optional units per column, per-row documentation, and collapsible unselected rows.

Data Table Cell

Insert key: 5  |  Icon: data table

A spreadsheet-style grid for tabular numeric data. Columns can be:

  • Input: user-entered values (with optional units)
  • Calculated: expressions evaluated for each row

Can generate interpolation or polynomial fit functions from the data, usable in other cells. Supports CSV import/export and per-column number formatting.

Piecewise Expression Cell

Insert key: 6  |  Icon: line chart

Defines a piecewise function: a parameter name plus a list of expression/condition pairs, with a final default (otherwise) case. Example:

f(x) = { x²     if x < 0
         { 2x    if x < 10
         { 100   otherwise

The result is a single named function available to the rest of the sheet.

System Solve Cell

Insert key: 7  |  Icon: IBM Watson Studio

Solves a system of simultaneous equations. Enter the equations and list the unknowns to solve for. Supports both exact (algebraic) and numerical solving with optional initial guesses. When multiple solutions exist, they are listed with a radio button to select which one to use downstream.

Code Cell

Insert key: 8  |  Icon: code brackets

Executes Python code directly in the browser via Pyodide. Define a function that can be called from math cells. Supports:

  • SymPy mode — return symbolic expressions
  • Numeric mode — return plain numbers
  • Access to all Pyodide-available packages
  • stdout capture displayed below the cell

The function name is entered in math notation and becomes callable in subsequent math cells.

Fluid Properties Cell

Insert key: 9  |  Icon: raindrop

Creates a thermodynamic property lookup function backed by the CoolProp library. Select a fluid, choose input/output properties (temperature, pressure, enthalpy, entropy, etc.), and give the function a name. Supports:

  • Pure fluids
  • Incompressible mixtures (with concentration)
  • Humid air (three-property inputs)
  • Custom fluid mixtures

The function is then callable in math cells like any other function.

Extreme Value Analysis (EVA) Cell

Insert key: 0  |  Icon: analytics bar chart

Finds the worst-case minimum and maximum of an output expression by evaluating it at all 2n combinations of parameter min/max bounds. Enter:

  • Query: the expression to evaluate (e.g., I=)
  • Parameters: one row per variable — name, min value, max value

Results display the computed minimum and maximum with units. Parameters can use plain numbers or values with units. Max 20 parameters.

Insert Sheet

Insert key: -  |  Icon: insert page

Not a cell — inserts a sheet boundary, splitting the document into multiple named sheets that share a single file but can be navigated independently.

DOCX Export (pandoc-wasm)

DOCX export uses pandoc-wasm — the full pandoc engine compiled to WebAssembly, the same engine as the server. Equations are converted to native Word OMML format (editable in Microsoft Word, not images). The WASM binary (~58 MB) is loaded on-demand the first time the user exports to DOCX.

PDF Export

Selecting PDF in standalone mode triggers window.print(). The app has extensive @media print styles across all cell components for correct layout.

Annotation Column

Math cells have an optional annotation column to the right for adding units descriptions, notes, or labels (e.g., "velocity", "kg/m³").

Usage

  • Click a math cell to reveal the annotation input to its right
  • Type a note — it saves automatically and persists with the sheet
  • Annotations are included in Markdown/DOCX export as *[annotation]*
  • Hidden on narrow screens (<500px) and rendered as plain text when printing

Extreme Value Analysis (EVA) Cell

Finds the worst-case min/max of an output expression by evaluating it at all 2n combinations of input parameter min/max bounds.

Usage

  1. Define parameters on the sheet (e.g., V = 10, R = 1000, I = V / R =)
  2. Insert an EVA cell via the toolbar (Analytics icon) or insert menu (key 0)
  3. Set the Query field to the expression to evaluate (e.g., I=)
  4. Add parameter rows: Parameter name, Min, and Max values (plain numbers or with units)
  5. Results display the overall min and max output values

How It Works

  • The query expression is rebuilt keeping EVA parameter symbols un-substituted
  • For n parameters, all 2n combinations of min/max bounds are evaluated
  • Each combination overrides the parameter symbol in parameter_subs and re-evaluates via get_evaluated_expression() with a fresh cache
  • Results show the min and max output across all combinations
  • Max 20 parameters (220 = ~1M combinations)

Plan: Extreme Value Analysis (EVA) Cell

Context

Users want worst-case min/max of an output expression by evaluating at all 2n combinations of parameter bounds. The EVA cell automates this with a parameter table (name, min, max) and a query field.

How It Works (Computation)

  1. Rebuild the query expression keeping EVA parameter symbols un-substituted
  2. For each combination (itertools.product): copy parameter_subs, override EVA parameter symbols, call get_evaluated_expression() with fresh cache
  3. Call get_result(), track numeric min and max
  4. Replace results[query_index] with ExtremeValueResult

Cell Model — ExtremeValueCell.svelte.ts

FieldTypePurpose
parameterFieldsMathField[] (type: "parameter")Parameter variable names
minFieldsMathField[] (type: "number")Min values
maxFieldsMathField[] (type: "number")Max values
queryFieldMathField (type: "math")Expression to evaluate
combinedMinFieldsMathField[]Internal: "param = min_value" assignments
combinedMaxFieldsMathField[]Internal: "param = max_value" assignments

Constraints

  • Max 20 parameters (enforced in frontend and backend)
  • Numeric results only — error if query produces symbolic output
  • Fresh expression_cache per combination to avoid stale results
  • Backwards compatible — extremeValueDefinitions defaults to []
